## Коллекция: `medical_records`

### Описание
Хранит историю медицинских записей пациентов. Каждая запись содержит информацию о приёме, диагнозе, назначениях и результатах.

| Поле | Тип | Описание | Обязательное |
|------|-----|---------|-------------|
| `_id` | ObjectId | Уникальный идентификатор документа (автоматически генерируется) | да |
| `code` | String | Уникальный код записи (REC-TIMESTAMP-RANDOM) | да |
| `patient_id` | String | Ссылка на ID пациента в PostgreSQL | да |
| `created_by` | String | Логин врача, создавшего запись | да |
| `title` | String | Заголовок записи (тип приёма/процедуры) | да |
| `description` | String | Подробное описание записи | да |
| `record_type` | String | Категория: checkup, surgery, test, prescription, diagnosis | нет |
| `severity` | String | Важность: low, medium, high, critical | нет |
| `tags` | Array[String] | Теги для категоризации и поиска | нет |
| `attachments` | Array[Object] | Массив прикреплённых документов и результатов | нет |
| `medications` | Array[Object] | Назначенные лекарства (embedded) | нет |
| `vital_signs` | Object | Показатели жизнедеятельности (embedded) | нет |
| `created_at` | Date | Дата создания записи | да |
| `updated_at` | Date | Дата последнего обновления | да |
| `is_deleted` | Boolean | Мягкое удаление записи | нет |

## Коллекция: `audit_logs`

### Описание
Ведёт историю всех изменений медицинских записей для обеспечения прозрачности и соответствия регуляциям.

| Поле | Тип | Описание | Обязательное |
| --- | --- | --- | --- |
| `_id` | ObjectId | Уникальный идентификатор лога (автоматически генерируется) | да |
| `record_code` | String | Код медицинской записи, к которой относится лог | да |
| `action` | String | Тип действия: `create`, `update`, `delete`, `restore` | да |
| `performed_by` | String | Логин пользователя (врача/администратора), выполнившего действие | да |
| `changes` | Object | Объект с деталями изменений (старые и новые значения) | нет |
| `timestamp` | Date | Точная дата и время совершения действия | да |
| `ip_address` | String | IP-адрес, с которого было произведено изменение | нет |
| `user_agent` | String | Информация о браузере/устройстве пользователя | нет |

## Embedded vs References

**Предложенная структура**:

medical_records
- References: patient_id -> patients table (PostgreSQL)
- References: created_by -> users table (PostgreSQL)
- Embedded:
   - medications[]
   - vital_signs{}
   - attachments[]

**Embedded**
- Врач всегда запрашивает все лекарства и показатели вместе с записью
- Обновление лекарств должно быть атомарным с записью
- Лекарства редко обновляются отдельно от записи
- Обычно у одной записи не очень много лекарст

**References**
- Пациенты и врачи - отдельные сущности с собственным жизненным циклом
- Один пациент может иметь сотни записей
- Информация о пациенте управляется независимо (в PostgreSQL)
- Упрощает изменение данных пациента без обновления всех записей
