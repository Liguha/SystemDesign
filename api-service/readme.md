# Домашнее задание 01: Документирование архитектуры в Structurizr

## 1. Вариант 20

Простой API Gateway для системы управления медицинскими записями. Все данные являются mock-данными, помеченными комментариями в коде (т.к. проектирование БД относится к следующим заданиям).

Реализованнная архитектура соответствует архитектуре из предыдущей работы (см. [../architecture/](../architecture/)).

## Доступные эндпоинты

- **POST /auth/login**: Выполняет вход пользователя в систему с предоставленными учетными данными.
- **DELETE /auth/logout**: Выполняет выход пользователя из системы.
- **POST /users**: Создает нового пользователя в системе.
- **GET /users/{login}**: Возвращает информацию о пользователе по его логину.
- **GET /users/search?mask=...**: Ищет пользователей по маске имени или логина.
- **POST /patients**: Регистрирует нового пациента в системе.
- **GET /patients/search?fio=...**: Ищет пациентов по ФИО.
- **GET /patients/{id}**: Возвращает информацию о пациенте по его ID.
- **POST /records**: Создает новую медицинскую запись для пациента.
- **GET /records/{code}**: Возвращает медицинскую запись по ее коду.
- **GET /patients/{id}/history**: Возвращает историю медицинских записей пациента по его ID.

## Сборка и запуск с Docker
```bash
cd docker
docker-compose up --build
```

API будет доступен на `http://localhost:8080`.

### Тестирование
```bash
bash tests/run.sh
```
